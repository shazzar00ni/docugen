# Security Policy

## Supported Versions

We currently support the following versions of DocuGen with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of DocuGen seriously. If you believe you have found a security vulnerability, please report it to us through responsible disclosure practices.

### How to Report

1. **Do not** create a public GitHub issue for security vulnerabilities
2. Email your report to: security@docugen.ai
3. Include as much detail as possible about the vulnerability:
   - Description of the issue
   - Steps to reproduce
   - Potential impact
   - Any known mitigations

### What to Expect

After you submit a report:

1. We will acknowledge receipt within 48 hours
2. Our security team will investigate and validate the report
3. We will keep you informed of our progress
4. Once fixed, we will credit you in our release notes (if you wish)

### Scope

This security policy applies to:

- The DocuGen React application
- Build tools and dependencies
- CI/CD pipelines
- Documentation

### Out of Scope

The following are not covered by this policy:

- Third-party services or integrations not maintained by DocuGen
- Social engineering attacks
- Physical security
- Issues in outdated versions

### Recognition

We appreciate responsible disclosure and will recognize contributors in our security acknowledgments (unless they prefer to remain anonymous).

### Thank You

Thank you for helping us keep DocuGen secure!
